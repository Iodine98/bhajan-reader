# Bhajan Reader

A browser-based Sanskrit chanting practice tool with word-by-word translation and real-time highlighting. Upload a `.bhajan` file and follow along phrase by phrase — the current word lights up across the Sanskrit and translation panels simultaneously.

## Features

- Word-by-word highlighting across Devanagari, IAST transliteration, and translation panels
- English and Dutch translations generated by Claude AI, cached to disk (and in-session)
- Microphone **onset detection** — advances the word cursor when it hears a syllable onset
- **Speech recognition** — matches spoken words to the text using fuzzy matching
- Operator + audience **display window** — real-time sync via BroadcastChannel, no server needed
- Light and dark theme, English and Dutch UI
- Hot-reload dev server, no build step, no npm

## Getting started

**Requirements**: Python 3.12+ and [uv](https://docs.astral.sh/uv/)

```bash
uv sync                      # install dependencies
uv run python3 server.py     # start server at http://localhost:8080
```

Open the settings modal (gear icon) and enter your [Claude API key](https://console.anthropic.com/). The key is stored in your browser's localStorage only — it is never sent to or stored by the server.

## Usage

### Loading a file

Click the upload button or drag-and-drop a `.bhajan` file onto the window. The app will fetch translations from Claude (or from the local cache if available), then render both panels.

### Keyboard navigation

| Key | Action |
|-----|--------|
| Space or → | Next word |
| ← | Previous word |
| ↓ or Enter | Next phrase |
| ↑ or Backspace | Previous phrase |

Keyboard shortcuts are suppressed when focus is inside a form field.

### Microphone input

Click the mic button to start listening. Choose a mode from the dropdown:

- **Onset Detection** (default) — advances the word cursor when the microphone detects a syllable onset. Configure the sensitivity threshold and refractory period in Settings.
- **Speech Recognition** — recognizes spoken words and jumps to the closest match in the text. Uses the browser's Web Speech API (Chrome/Edge) or falls back to server-side transcription (Firefox).

### Display window

Open Settings → "Open Display Window" to launch a second fullscreen window for projection or a second screen. It mirrors the operator window in real time and scales the text larger for audience viewing. Keep the operator window open — the display window only receives updates from it.

## .bhajan file format

Plain UTF-8 text with a metadata header and verse sections:

```
# title: Gayatri Mantra
# composer: Vishwamitra (from Rigveda 3.62.10)
---
[verse:1]
ॐ भूर्भुवः स्वः
तत्सवितुर्वरेण्यं
भर्गो देवस्य धीमहि
धियो यो नः प्रचोदयात्
---
```

- `# key: value` lines before `---` are metadata (any keys accepted)
- `---` toggles the body section on/off
- `[name]` markers start a verse or section (e.g. `[verse:1]`, `[chorus]`)
- Each line under a marker is one **phrase** — the atomic unit for navigation and highlighting
- Blank lines are ignored

See `examples/` for complete sample files.

## Running tests

Requires the server to be running (`uv run python3 server.py`).

```bash
uv run pip install playwright pytest
playwright install chromium
pytest tests/ -v
```

## Architecture

No framework, no bundler. The frontend is vanilla ES6 modules in `modules/` served directly by `server.py`. `app.js` is the entry point. The Python server also proxies the Anthropic API (to avoid CORS), saves translation JSON to `translations/` on disk, and watches files for hot-reload via SSE.

For a detailed architecture reference including the data flow, module descriptions, and caching layers, see [CLAUDE.md](CLAUDE.md).
